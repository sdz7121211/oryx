68  /**
69   * <p>This class will run one generation of the Computation Layer. It is essentially a client of Hadoop
70   * and starts and monitors a series of MapReduce jobs that complete the Computation Layer's work.</p>
71   *
72   * <p>This is an overview of the sequence of MapReduce jobs that can run as part of the ALS pipeline.
73   * While it is one flow, there are several distinct concerns inside of it, some optional. The core
74   * flow is presented first, followed by notes about secondary or optional aspects of the flow. These
75   * are presented in order of execution although in many cases some jobs run in parallel with others.</p>
76   *
77   * <p>In all cases, generation N's data is found at HDFS path {@code /[bucket]/[N]/} (where N is
78   * padded to 5 digits so that lexical order equals numeric order). Locations are relative to this
79   * unless otherwise specified.</p>
80   *
81   * <ul>
82   * <li>"varlong" refers to the representation {@code long}</li>
83   * <li>"varint" refers to {@code int}</li>
84   * <li>"idvalue" refers to {@code IDValue}</li>
85   * <li>"floatarray" refers to {@code float[]}</li>
86   * <li>"floatmap" refers to {@code FastByIDFloatMap}</li>
87   * <li>"idset" refers to {@code FastIDSet}</li>
88   * </ul>
89   *
90   *
91   * <h2>Core Steps</h2>
92   *
93   * <h3>SplitTestStep</h3>
94   *
95   * <p>New input is always found in generation N under {@code inbound/}, in (compressed) CSV text files.
96   * This step splits the data into test and train data (if {@code model.test-set-fraction} is nonzero).
97   * These go into {@code test/} and {@code train/}, respectively, unmodified.</p>
98   *
99   * <h3>MergeNewOldStep</h3>
100  *
101  * <p>Merges all data from last generation (including the last generation's test data)
102  * with new input in the current generation. The last generation's
103  * data exists in generation N-1 under {@code input/} and {@code test/} (because the test data was not part of the
104  * input to the prior generation). It encodes user,item,value tuples as
105  * user-(item,value) key/value pairs, in particular as varlong mapped to idvalue objects.</p>
106  *
107  * <p>This step merges old and new values by adding them. Here is where 'decay' of old data takes place too.</p>
108  *
109  * <p>Output goes to generation N's {@code input/}, and this is what drives all further computation in
110  * the generation.</p>
111  *
112  * <h3>ToUserVectorsStep / ToItemVectorsStep</h3>
113  *
114  * <p>These simply transform the output above into a mapping from user to all item/value pairs as a single
115  * floatmap object -- user vectors if you like -- and likewise for item vectors. The output goes to
116  * {@code userVectors/} and {@code itemVectors/} respectively.</p>
117  *
118  * <h3>CollectKnownItemsStep</h3>
119  *
120  * <p>Outputs user IDs mapped to a set of item IDs that the user ID has interacted with in the data set.
121  * The output is in text format, in {@code knownItems/} and is for consumption by the Serving Layer.</p>
122  *
123  * <h3>InitialYStep</h3>
124  *
125  * <p>This process outputs an initial Y matrix. It will optionally reuse the final {code Y/} output from
126  * last generation, N-1. For any items that have no vector yet (which might be all of them on a fresh run)
127  * it will choose random unit vectors (chosen with kmeans++-style preference for dissimilar vectors).
128  * Output goes to {@code tmp/iterations/0/Y}, and is encoded as varlong mapped to floatarray.</p>
129  *
130  * <h3>PopularUserStep / PopularItemStep</h3>
131  *
132  * <p>Slightly misnamed, as they actually output the set of items that a given user interacts with in the
133  * data, and vice versa for items. The output is encoded as varlong mapped to idset, and the output goes
134  * to {@code popularUsersByItemPartition} and {@code popularItemsByUserPartition} respectively.. This is
135  * used later to cleverly side-load only the part of the X or Y matrix that will actually be used by
136  * a reducer.</p>
137  *
138  * <h3>RowStep</h3>
139  *
140  * <p>The heart of the ALS iteration, and the most important step in the flow. This is the one that is
141  * iterated many times. It will alternately compute X from Y, then Y from X. In the first case, for
142  * iteration i, input is from {@code tmp/iterations/[i-1]/Y} and output goes to
143  * {@code tmp/iterations/[i]/X}. In the second case, it's {@code tmp/iterations/[i]/X} going to
144  * {code tmp/iterations/[i]/Y}. Input and output are rows of a matrix: varlong mapped to floatarray.</p>
145  *
146  * <p>The core ALS computation here is fairly straightforward, but, there is complexity around
147  * side-loading the X or Y matrix, rather than literally joining the matrix on the input, which would
148  * be far too slow. To conserve memory, only keys that are expected to appear in the input to the
149  * particular reducer will be loaded.</p>
150  *
151  * <h3>PublishXStep / PublishYStep</h3>
152  *
153  * <p>Simple map-only jobs that publish the X or Y matrix, respectively, from the final iteration, in a
154  * form that's consumable by the Serving Layer. The output is compressed text format. See the source
155  * code for exact format. It is otherwise the same matrix as computed by the iterations. The output
156  * goes to {@code X/} and {@code Y/} respectively.</p>
157  *
158  * <h2>Recommend Steps</h2>
159  *
160  * <h3>DistributeRecommendWorkStep</h3>
161  *
162  * <p>This first step effectively joins data from {@code X/} (user-feature matrix) and {@code knownItems/},
163  * since both are needed to recommend for a user. It writes the combined values as varlong mapped to
164  * {@code IDAndKnownItemsAndFloatArrayWritable}. The output goes to {@code tmp/distributeRecommend/}.
165  * This is necessary as its own step because the shuffle/combiner will need to be used differently in
166  * the next step.</p>
167  *
168  * <h3>RecommendStep</h3>
169  *
170  * <p>This computes recommendations given output above. The item-features matrix, Y, is also needed.
171  * It is side-loaded cleverly as in {@code RowStep}. Each reducer makes N recommendations per user when
172  * N recommendations are desired. They will be condensed into N overall in the next step. The output
173  * goes to {@code tmp/partialRecommend} and is encoded as varlong (user ID) mapped to a series of idvalue
174  * objects representing top recommended items.</p>
175  *
176  * <h3>CollectRecommendStep</h3>
177  *
178  * <p>This stage simply takes output above and outputs final N recommendations for each user. Output
179  * is conceptually similar, but in text format, and goes to {@code recommend/}.
180  *
181  *
182  * <h2>Similarity Steps</h2>
183  *
184  * <h3>DistributeSimilarWorkStep</h3>
185  *
186  * <p>Quite analogous to {@code DistributeRecommendWorkStep} and {@code RecommendStep} combined. Because
187  * there is no join needed, these two tasks can be accomplished in a single job.</p>
188  *
189  * <h3>SimilarStep</h3>
190  *
191  * <p>Quite analogous to {@code CollectRecommendStep}.</p>
192 
193  * @author Sean Owen
194  */
195 public final class ALSDistributedGenerationRunner extends DistributedGenerationRunner {

199   @Override
200   protected List<DependsOn<Class<? extends JobStep>>> getPreDependencies() {
201     List<DependsOn<Class<? extends JobStep>>> preDeps = Lists.newArrayList();
202     preDeps.add(DependsOn.<Class<? extends JobStep>>first(MergeIDMappingStep.class));
203     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(MergeNewOldStep.class, SplitTestStep.class));
204     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(ToUserVectorsStep.class, MergeNewOldStep.class));
205     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(ToItemVectorsStep.class, MergeNewOldStep.class));
206     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(CollectKnownItemsStep.class, ToUserVectorsStep.class));
207     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(InitialYStep.class, ToItemVectorsStep.class));
208     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(PopularUserStep.class, ToUserVectorsStep.class));
209     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(PopularItemStep.class, ToItemVectorsStep.class));
210     return preDeps;
211   }
212 
213   @Override
214   protected List<DependsOn<Class<? extends JobStep>>> getIterationDependencies() {
215     List<DependsOn<Class<? extends JobStep>>> iterationsDeps = Lists.newArrayList();
216     iterationsDeps.add(DependsOn.<Class<? extends JobStep>>first(RowStep.class));
217     return iterationsDeps;
218   }

220   @Override
221   protected List<DependsOn<Class<? extends JobStep>>> getPostDependencies() {
222     Config config = ConfigUtils.getDefaultConfig();
223     List<DependsOn<Class<? extends JobStep>>> postDeps = Lists.newArrayList();
224     postDeps.add(DependsOn.<Class<? extends JobStep>>first(PublishXStep.class));
225     postDeps.add(DependsOn.<Class<? extends JobStep>>first(PublishYStep.class));
226     if (config.getBoolean("model.recommend.compute")) {
227       postDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(RecommendStep.class,
228                                                                       DistributeRecommendWorkStep.class));
229       postDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(CollectRecommendStep.class, RecommendStep.class));
230     }
231     if (config.getBoolean("model.item-similarity.compute")) {
232       postDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(SimilarStep.class, DistributeSimilarWorkStep.class));
233     }
234     return postDeps;
235   }

250   @Override
251   protected void runOneIteration(int iterationNumber, List<Collection<Class<? extends JobStep>>> iterationSchedule)
252       throws InterruptedException, JobException, IOException {
253     for (Collection<Class<? extends JobStep>> iterationStepClasses : iterationSchedule) {
254       runSchedule(iterationStepClasses, buildConfig(iterationNumber, true)); // X

261   @Override
262   protected boolean areIterationsDone(int iterationNumber) throws IOException {
263 
264     // First maybe output MAP
265     Store store = Store.get();
266     String iterationsPrefix = Namespaces.getIterationsPrefix(getInstanceDir(), getGenerationID());
267 
268     String mapKey = iterationsPrefix + iterationNumber + "/MAP";
269     if (store.exists(mapKey, true)) {
270       double map = Double.parseDouble(store.readFrom(mapKey).readLine());
271       log.info("Mean average precision estimate: {}", map);
272     }
273 
274     if (iterationNumber < 2) {
275       return false;
276     }
277 
278     Config config = ConfigUtils.getDefaultConfig();
279     int maxIterations = config.getInt("model.iterations.max");
280 
281     if (maxIterations > 0 && iterationNumber >= maxIterations) {
282       log.info("Reached iteration limit");
283       return true;
284     }
285 
286 
287     String previousConvergenceKey = iterationsPrefix + (iterationNumber-1) + "/Yconvergence/";
288     String currentConvergenceKey = iterationsPrefix + iterationNumber + "/Yconvergence/";
289     Preconditions.checkState(store.exists(currentConvergenceKey, false),
290                              "No convergence samples in current iteration");
291     if (!store.exists(previousConvergenceKey, false)) {
292       // Previous iteration was deleted already because next iteration failed to start; continue
293       return false;
294     }
295 
296     LongObjectMap<LongFloatMap> previousEstimates = readUserItemEstimates(previousConvergenceKey);
297     LongObjectMap<LongFloatMap> estimates = readUserItemEstimates(currentConvergenceKey);
298     Preconditions.checkState(estimates.size() == previousEstimates.size(),
299                              "Estimates and previous estimates not the same size: %s vs %s",
300                              estimates.size(), previousEstimates.size());
301 
302     DoubleWeightedMean averageAbsoluteEstimateDiff = new DoubleWeightedMean();
303     for (LongObjectMap.MapEntry<LongFloatMap> entry : estimates.entrySet()) {
304       long userID = entry.getKey();
305       LongFloatMap itemEstimates = entry.getValue();
306       LongFloatMap previousItemEstimates = previousEstimates.get(userID);
307       Preconditions.checkState(itemEstimates.size() == previousItemEstimates.size(),
308                                "Number of estaimtes doesn't match previous: {} vs {}",
309                                itemEstimates.size(), previousItemEstimates.size());
310       for (LongFloatMap.MapEntry entry2 : itemEstimates.entrySet()) {
311         long itemID = entry2.getKey();
312         float estimate = entry2.getValue();
313         float previousEstimate = previousItemEstimates.get(itemID);
314         // Weight, simplistically, by newValue to emphasize effect of good recommendations.
315         // But that only makes sense where newValue > 0
316         if (estimate > 0.0f) {
317           averageAbsoluteEstimateDiff.increment(FastMath.abs(estimate - previousEstimate), estimate);
318         }
319       }
320     }
321 
322     double convergenceValue;
323     if (averageAbsoluteEstimateDiff.getN() == 0) {
324       // Fake value to cover corner case
325       convergenceValue = FastMath.pow(2.0, -(iterationNumber + 1));
326       log.info("No samples for convergence; using artificial convergence value: {}", convergenceValue);
327     } else {
328       convergenceValue = averageAbsoluteEstimateDiff.getResult();
329       log.info("Avg absolute difference in estimate vs prior iteration: {}", convergenceValue);
330       if (!Doubles.isFinite(convergenceValue)) {
331         log.warn("Invalid convergence value, aborting iteration! {}", convergenceValue);
332         return true;
333       }
334     }
335 
336     double convergenceThreshold = config.getDouble("model.iterations.convergence-threshold");
337     if (convergenceValue < convergenceThreshold) {
338       log.info("Converged");
339       return true;
340     }
341     return false;
342   }

344   private static LongObjectMap<LongFloatMap> readUserItemEstimates(String convergenceSamplePrefix)
345       throws IOException {
346     log.info("Reading estimates from {}", convergenceSamplePrefix);
347     LongObjectMap<LongFloatMap> userItemEstimate = new LongObjectMap<LongFloatMap>();
348     Store store = Store.get();
349     for (String prefix : store.list(convergenceSamplePrefix, true)) {
350       for (CharSequence line : new FileLineIterable(store.readFrom(prefix))) {
351         String[] tokens = DelimitedDataUtils.decode(line, ',');
352         long userID = Long.parseLong(tokens[0]);
353         long itemID = Long.parseLong(tokens[1]);
354         float estimate = Float.parseFloat(tokens[2]);
355         LongFloatMap itemEstimate = userItemEstimate.get(userID);
356         if (itemEstimate == null) {
357           itemEstimate = new LongFloatMap();
358           userItemEstimate.put(userID, itemEstimate);
359         }
360         itemEstimate.put(itemID, estimate);
361       }
362     }
363     return userItemEstimate;
364   }

366   @Override
367   protected void doPost() throws IOException {
368     log.info("Loading X and Y to test whether they have sufficient rank");
369 
370     String generationPrefix = Namespaces.getInstanceGenerationPrefix(getInstanceDir(), getGenerationID());
371     String xPrefix = generationPrefix + "X/";
372     String yPrefix = generationPrefix + "Y/";
373 
374     Store store = Store.get();
375     if (!doesXorYHasSufficientRank(xPrefix) || !doesXorYHasSufficientRank(yPrefix)) {
376       log.warn("X or Y does not have sufficient rank; deleting this model and its results");
377       store.recursiveDelete(xPrefix);
378       store.recursiveDelete(yPrefix);
379       store.recursiveDelete(generationPrefix + "recommend/");
380       store.recursiveDelete(generationPrefix + "similarItems/");
381     } else {
382       log.info("X and Y have sufficient rank");
383       File tempModelDescriptionFile = File.createTempFile("model-", ".pmml.gz");
384       tempModelDescriptionFile.deleteOnExit();
385       ALSModelDescription modelDescription = new ALSModelDescription();
386       modelDescription.setKnownItemsPath("knownItems");
387       modelDescription.setXPath("X");
388       modelDescription.setYPath("Y");
389       modelDescription.setIDMappingPath("idMapping");
390       ALSModelDescription.write(tempModelDescriptionFile, modelDescription);
391       store.upload(generationPrefix + "model.pmml.gz", tempModelDescriptionFile, false);
392       IOUtils.deleteRecursively(tempModelDescriptionFile);
393     }
394 
395     super.doPost();
396   }

398   private static boolean doesXorYHasSufficientRank(String xOrYPrefix) throws IOException {
399     double[][] transposeTimesSelf = null;
400     Store store = Store.get();
401     for (String xOrYFilePrefix : store.list(xOrYPrefix, true)) {
402       for (String line : new FileLineIterable(store.readFrom(xOrYFilePrefix))) {
403         int tab = line.indexOf('\t');
404 
405         float[] elements = DataUtils.readFeatureVector(line.substring(tab + 1));
406         int features = elements.length;
407 
408         if (transposeTimesSelf == null) {
409           transposeTimesSelf = new double[features][features];
410         }
411 
412         for (int i = 0; i < features; i++) {
413           double rowFactor = elements[i];
414           double[] transposeTimesSelfRow = transposeTimesSelf[i];
415           for (int j = 0; j < features; j++) {
416             transposeTimesSelfRow[j] += rowFactor * elements[j];
417           }
418         }
419 
420       }
421       
422       // Hacky but efficient and principled: if a portion of the matrix times itself is non-singular,
423       // it's all but impossible that it would be singular when the rest was loaded
424       
425       if (transposeTimesSelf != null && 
426           transposeTimesSelf.length != 0 && 
427           MatrixUtils.isNonSingular(new Array2DRowRealMatrix(transposeTimesSelf))) {
428         return true;
429       } else {
430         log.info("Matrix is not yet proved to be non-singular, continuing to load...");
431       }
432     }
433     return false;
434   }
